결과: `job_submitted` 이벤트에서 molecule_name, method, basis_set이 **항상 빈 문자열로 upsert**됩니다. 이후 `result` 이벤트가 오면 `deepMerge`로 덮어써지므로 최종 결과에는 영향이 없지만, **히스토리에서 job이 완료되기 전까지 빈 이름으로 표시**됩니다.

수정:

```javascript
case "job_submitted": case "submitted": case "queued":
case "job_created": case "job_queued":
    var jid = jobId || state.activeJobId;
    var jobSnap = msg.job || msg;  // job snapshot이 msg.job에 있을 수 있음
    if (!jid && jobSnap.job_id) jid = jobSnap.job_id;
    if (!jid) break;
    state.activeJobId = jid;
    App.upsertJob({
        job_id: jid,
        status: "queued",
        submitted_at: Date.now() / 1000,
        updated_at: Date.now() / 1000,
        user_query: state.lastUserInput,
        molecule_name: jobSnap.molecule_name || msg.molecule_name || "",
        method: jobSnap.method || msg.method || "",
        basis_set: jobSnap.basis_set || msg.basis_set || "",
    });
    // ...
    break;
```

**버그 41 (신규): `handleServerEvent`의 `result` case에서도 같은 문제**

```javascript
case "result":
    var result = msg.result || msg.results || msg.data || msg.output || msg.computation || null;
```

서버가 보내는 형식:

```python
await _ws_send(
    websocket,
    "result",
    session_id=session_id,
    job=terminal,
    result=result,         # ← 여기에 있음
    summary=_result_summary(result),
)
```

`msg.result`가 올바르게 추출됩니다. 하지만:

```javascript
App.upsertJob({
    job_id: rjid, status: "completed", result: result,
    molecule_name: result.molecule_name || result.molecule || "",
    method: result.method || "",
    basis_set: result.basis_set || "",
```

`result.basis_set`는 백엔드가 `result.basis`로 보내므로 항상 빈 문자열. `result.molecule_name`은 백엔드에서 `structure_name`으로 보내므로... 확인:

```python
# _make_base_result
result = {
    "structure_name": _safe_str(structure_name, "custom"),
    "structure_query": _safe_str(structure_name, "custom"),
```

하지만 `_normalize_result_contract` (compute.py)에서:

```python
out.setdefault("structure_name", payload.get("structure_query") or payload.get("structure_name"))
```

`structure_name`은 설정됩니다. 프론트엔드에서 `result.molecule_name`은 없고 `result.structure_name`이 있습니다.

```javascript
molecule_name: result.molecule_name || result.molecule || "",
```

`result.molecule_name`은 undefined이고 `result.molecule`도 undefined이므로 빈 문자열. **히스토리에서 완료된 job도 분자 이름이 빈 문자열**.

app.js의 `getJobDisplayName`에서:

```javascript
var molName = job.molecule_name || job.molecule ||
    (job.result && (job.result.structure_name || job.result.structure_query)) ||
    (job.payload && (...));
```

`job.result.structure_name`을 fallback으로 읽으므로 **히스토리에서는 표시됩니다**. 하지만 `job.molecule_name` 자체는 항상 빈 문자열.

이건 **심각한 버그는 아니지만 데이터 정합성 문제**입니다. upsert에서 `structure_name`을 직접 저장하는 게 맞습니다:

```javascript
molecule_name: result.structure_name || result.molecule_name || result.molecule || "",
method: result.method || "",
basis_set: result.basis || result.basis_set || "",
```

### `index.html` 재검토

**이슈 43 (신규): `makeSessionId`가 collision 가능**

```javascript
function makeSessionId() {
  var ts = Date.now().toString(36);
  var r = Math.random().toString(36).substring(2, 8);
  return "qcviz-" + ts + "-" + r;
}
```

`Math.random()`은 암호학적으로 안전하지 않고, 같은 밀리초에 두 탭을 열면 `ts`가 같습니다. `r` 부분은 6자리 base36 (~2.2억 조합)이므로 충돌 확률은 극히 낮지만, `crypto.getRandomValues`를 쓰면 더 안전합니다. 실용적으로는 문제없음.
