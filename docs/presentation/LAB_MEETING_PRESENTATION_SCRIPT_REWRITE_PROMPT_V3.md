# Lab Meeting Script Rewrite Prompt V3

## 목적

기존 한국어 발표 대본 V2를 연구실 청중 설득력 기준으로 다시 쓴다.
이번 버전은 "문장을 조금 다듬는 작업"이 아니라, 청중이 실제로 궁금해하는 질문 순서에 맞게 발표 구조와 설명 강도를 재설계하는 작업이다.

## 청중

- 실험 화학자
- 계산 화학자
- LLM, MCP, 소프트웨어 아키텍처에 익숙하지 않은 연구실 구성원

## 반드시 해결해야 할 문제

1. 메타 발화 제거
- 발표자가 속으로 생각해야 할 말, 예를 들어 "이 슬라이드는 이런 역할이다", "여기서 모드를 바꿔야 한다", "이 슬라이드가 중요하다" 같은 문장은 발표 대본에서 제거한다.
- 청중에게 실제로 들려줄 설명만 남긴다.

2. LLM 설명 강화
- "질문을 번역한다"는 표현을 추상적으로 쓰지 말고, 실제로 무엇을 뽑아내는지 설명한다.
- 예: 분자명, 계산 종류, 후속질문 맥락, 부족한 정보, 필요한 계산 결과 형식.
- 청중이 "그럼 이건 그냥 말 잘하는 모델 아닌가?"라고 생각할 여지를 줄인다.

3. 할루시네이션 우려를 정면으로 처리
- LLM 단독 답변은 왜 위험한지 먼저 설명한다.
- "가짜 값을 말할 수 있다", "계산한 척할 수 있다", "그럴듯한 설명이 실제 계산 결과를 대체할 수 없다"는 점을 분명히 한다.
- 그 다음에 MCP와 실제 계산 엔진 호출 구조가 어떻게 이 문제를 줄이는지 설명한다.

4. MCP를 완전히 처음 듣는 청중 기준으로 설명
- MCP를 바로 정의하지 말고, 먼저 다른 분야 사례로 감을 잡게 한다.
- 예: PowerPoint MCP, Blender MCP, 문서 도구 MCP, 코드 실행 도구 MCP.
- 그 다음 "그런데 양자화학에는 이런 연결 구조가 거의 없었다"는 공백을 제시한다.
- 마지막으로 "우리는 그 공백을 선점하려 한다"는 연구적 의미로 연결한다.

5. PySCF의 위치를 더 강하게 설명
- PySCF는 단순한 라이브러리가 아니라, 실제 계산 결과를 만들어내는 과학적 백엔드라는 점을 반복해서 강조한다.
- "PySCF가 없으면 이 시스템은 계산 도구가 아니라 계산 흉내를 내는 인터페이스에 불과하다"는 메시지를 분명히 한다.

6. 앞부분은 중요성과 구현 원리에 집중
- 데모 전략, 대화 예시, 화면 예시는 뒤로 보낸다.
- 앞 70%는 다음 질문에 답해야 한다:
  - 왜 필요한가?
  - 왜 지금 가능한가?
  - 왜 이게 단순 챗봇이 아닌가?
  - 어떻게 실제 계산까지 연결되는가?

7. 슬라이드 존재 이유를 엄격히 판단
- 슬라이드 하나당 하나의 설득 포인트만 둔다.
- 힘이 약한 슬라이드는 삭제하거나 다른 슬라이드에 병합한다.
- "현재 상태", "아직 필요한 검증"처럼 설명은 필요하지만 독립 장으로 힘이 약한 내용은 임팩트 슬라이드와 합친다.

8. 실용적 임팩트 강화
- 연구실에서 왜 중요한지 더 강하게, 더 구체적으로 말한다.
- 실험자의 자율성 확대
- 계산 전문가 병목 완화
- 실험-계산 피드백 루프 단축
- 교육적 효과
- 연구실 계산 문화 변화

9. 후속 연구 방향 구체화
- publication plan은 단순 논문화 언급이 아니라, 어디까지 확장할 수 있는지 보여줘야 한다.
- 예: cluster DFT, surface DFT, periodic DFT, AIMD, 더 복잡한 다성분계.

## 스스로에게 던질 질문

- 이 슬라이드는 청중에게 새로운 이해를 주는가, 아니면 발표자 메모에 가까운가?
- 화학자가 "LLM이 계산한 게 아니면 대체 뭐를 한 거지?"라고 물었을 때 이 대본은 바로 답할 수 있는가?
- "이거 할루시네이션 아니야?"라는 질문을 발표 중간에 선제적으로 막는가?
- MCP를 모르는 사람이 "아, 결국 LLM이 외부 계산 도구를 제대로 부리게 해주는 규칙이구나"라고 이해할 수 있는가?
- 양자화학에서 왜 이 구조가 특히 의미가 큰지 충분히 말했는가?
- 슬라이드 전반부가 데모가 아니라 필요성, 구조, 신뢰성 설명에 충분한 시간을 쓰고 있는가?
- 실험 화학자가 "이거 우리 연구실에서 진짜 유용하겠다"라고 느낄 포인트가 명확한가?

## 최종 산출물 요구사항

- 한국어 발표 대본 V3
- 슬라이드 수는 24~28장 수준으로 재구성
- 실제 발표용 문장만 사용
- MCP, LLM, PySCF, hallucination 구분을 핵심 설득 축으로 삼기
- 데모 파트는 마지막 2~3장만 사용
- 전체 톤은 자신감 있고 명료하되 과장된 수치 주장은 피하기
