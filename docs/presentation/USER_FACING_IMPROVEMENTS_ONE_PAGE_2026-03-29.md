# 사용자 체감 개선점 한 장 요약

## 핵심 메시지

이번 고도화의 핵심은 계산 기능을 새로 많이 붙인 것이 아니라, **사용자가 실제로 입력하는 자연어를 더 정확히 이해하고, 덜 엉뚱하게 반응하며, 이전 문맥을 더 자연스럽게 이어받도록 만든 것**입니다.

## 이전에 사용자 입장에서 불편했던 점

- `베 ㄴ젠`처럼 띄어쓰기나 한글 자모가 섞이면 분자명을 제대로 못 알아듣는 경우가 있었습니다.
- `니트로 벤젠`처럼 한국어 치환기 표현은 한 분자로 묶지 못하고 구조 탐색에 실패할 수 있었습니다.
- 구조를 못 찾았을 때 `water` 같은 무관한 기본 후보로 튀는 fallback이 사용자 신뢰를 떨어뜨렸습니다.
- 이전 계산 문맥이 있는데도 `ESP도 그려줘` 같은 후속 질문을 새 요청처럼 처리하는 경우가 있었습니다.
- 반대로 문맥이 없는 상태에서 분석 요청만 들어오면, 시스템이 엉뚱한 구조를 추측할 위험이 있었습니다.
- `EMIM+ TFSI-` 같은 이온쌍 입력도 단일 분자처럼 다뤄질 가능성이 있었습니다.

## 이번에 달라진 점

- 깨진 한글 입력이나 붙여 쓴 입력을 더 잘 복원해 `benzene`, `nitrobenzene` 같은 canonical name으로 연결합니다.
- `Biphenyl`, `benzoic acid`, `아세트산`, `메틸아민` 같은 단일 분자 요청은 불필요한 후보 선택 없이 바로 계산으로 들어가는 경우가 많아졌습니다.
- 구조 해석에 실패했을 때 raw 문장 전체나 무관한 추천 분자를 밀어 넣지 않고, 더 그럴듯한 후보만 제시하도록 정리했습니다.
- 같은 세션 안에서는 `HOMO → ESP → optimization`처럼 이어지는 후속 질문에서 이전 구조를 재사용합니다.
- 문맥이 없을 때는 무리하게 추측하지 않고, 어떤 분자인지 먼저 명확히 묻도록 clarification 흐름을 정리했습니다.
- 이온쌍과 salt 입력은 단일 구조명으로 찌그러뜨리지 않고, 양이온/음이온 조합으로 유지한 채 처리합니다.

## 사용자가 실제로 느끼는 변화

- 시스템이 **덜 엉뚱하게 알아듣습니다.**
- 시스템이 **덜 많이 되묻습니다.**
- 시스템이 **이전 질문 맥락을 더 잘 이어받습니다.**
- 구조를 애매하게 이해했을 때도 **잘못 계산하기보다 안전하게 확인하는 방향**으로 동작합니다.
- 결과적으로 사용자는 계산 도구를 다루는 느낌보다, **연구 질문을 이어서 대화하는 느낌**에 더 가까운 경험을 하게 됩니다.

## 발표용 Take-home Message

> 이번 개선의 본질은 “더 똑똑한 답변”이 아니라,  
> **더 정확한 구조 해석, 더 안전한 fallback, 더 자연스러운 문맥 유지**입니다.
