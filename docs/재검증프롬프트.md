### 응답 단계 3 — 자체 검증 (★ 이 단계를 절대 생략하지 마라)

수정 완료 후, 반드시 아래 **5가지 검증 항목**을 하나씩 실행하고 결과를 출력한다:

```
===== ✅ 검증 체크리스트 (수정요구사항 N번) =====

[체크 1] 파일 존재 확인
→ 수정한 파일을 다시 열어서 실제로 변경이 저장되었는지 확인.
→ 결과: PASS / FAIL + 근거

[체크 2] 구문 오류 확인
→ Python 파일: `python -m py_compile <파일경로>` 또는 AST parse
→ JS 파일: 기본 구문 검사 (괄호/중괄호 매칭, 세미콜론 등)
→ 결과: PASS / FAIL + 근거

[체크 3] 수정 범위 정확성
→ .md에 명시된 변경 사항이 빠짐없이 모두 반영되었는가?
→ 누락된 항목이 있으면 즉시 추가 수정 후 재검증.
→ 결과: PASS(전체 반영) / PARTIAL(누락 항목 목록) / FAIL

[체크 4] 기존 기능 훼손 여부
→ 변경하지 않아야 할 코드를 실수로 건드리지 않았는가?
→ 수정 대상 함수의 시그니처·반환값이 호출부와 여전히 호환되는가?
→ import 경로, 변수명 등이 다른 파일에서 참조하는 이름과 일치하는가?
→ 결과: PASS / FAIL + 근거

[체크 5] 엣지케이스·부작용
→ 수정으로 인해 새로운 버그가 생길 가능성을 1~2개 시나리오로 점검.
→ 결과: PASS(부작용 없음) / WARNING(잠재 이슈 설명)

===== 종합 판정: ✅ ALL PASS / ⚠️ PARTIAL / ❌ FAIL =====
```

### 응답 단계 4 — 완료 선언 또는 재수정

- 종합 판정이 **ALL PASS**이면: `"✅ 수정요구사항 N번 완료. 다음 파일(N+1번) 진행하겠습니다."` 라고 선언.
- 종합 판정이 **PARTIAL 또는 FAIL**이면: 즉시 문제를 수정하고 **체크 1~5를 다시 전부 실행**한다. ALL PASS가 될 때까지 반복.

3. **한국어 응답**: 모든 설명, 검증 결과, 선언을 한국어로 작성.
